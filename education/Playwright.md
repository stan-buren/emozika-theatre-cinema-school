# 🎭 Playwright: Твой Цифровой Тестировщик

## Что это?
**Playwright** — это инструмент для E2E (End-to-End) тестирования.
Представь, что ты нанял человека, который после каждого твоего изменения заходит на сайт и проверяет:
1.  "А кнопка нажимается?"
2.  "А текст правильный?"
3.  "А картинка загрузилась?"

Playwright — это этот человек, только это робот, и он делает это за 2 секунды.

## Аналогия с CS50
В CS50 был `check50`.
Ты отправлял код, и `check50` прогонял набор тестов: "Если ввести 5, программа выведет 10?".

Playwright — это `check50` для твоего сайта.

## Примеры кода
Вот как выглядит тест (`tests_integration/example.spec.ts`):

```typescript
import { test, expect } from '@playwright/test';

// Тест: "Проверка заголовка"
test('имеет правильный заголовок', async ({ page }) => {
  // 1. Робот, открой страницу
  await page.goto('http://localhost:4321/');

  // 2. Найди элемент h1 и проверь, что в нем текст "Эмоцика"
  await expect(page.locator('h1')).toHaveText(/Эмоцика/);
});
```

Если ты сломаешь заголовок, Playwright покраснеет и скажет: "Ожидал 'Эмоцика', получил 'Ошибка'".

## Зачем нам это?
Когда проект станет большим, ты будешь бояться менять код, чтобы не сломать что-то в другом месте.
Тесты дают тебе **уверенность**. Ты меняешь код, запускаешь `npm test`, видишь зеленые галочки — и спокойно заливаешь изменения.
