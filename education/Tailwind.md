# 🎨 Tailwind CSS: Стили без CSS-файлов

## Что это?
**Tailwind** — это библиотека "утилитных" классов.
Вместо того чтобы придумывать имя класса `.message-box` и описывать его в CSS файле:
```css
/* index.css */
.message-box {
    display: flex;
    padding: 1rem;
    background-color: black;
    color: white;
}
```

Ты просто пишешь готовые классы прямо в HTML:
```html
<div class="flex p-4 bg-black text-white">
  ...
</div>
```

## Зачем? (Аналогия с CS50)
Вспомни Си. У тебя есть функции из стандартной библиотеки (`printf, get_string`). Тебе не нужно писать их реализацию каждый раз, ты просто их *вызываешь*.

Tailwind — это "стандартная библиотека" для CSS.
*   `flex` = `display: flex;`
*   `bg-black` = `background-color: black;`
*   `p-4` = `padding: 1rem;`

## Плюсы подхода
1.  **Скорость:** Не нужно переключаться между `.html` и `.css` файлами.
2.  **Нет конфликтов имен:** Ты никогда не сломаешь верстку, случайно назвав два блока `.container`.
3.  **Малый вес:** При сборке (`npm run build`) Tailwind выкинет все классы, которые ты *не использовал*, и оставит крошечный CSS файл.

## Важное замечание
Сначала это выглядит уродливо (`class="flex items-center justify-center min-h-screen bg-gray-100..."`).
Но через неделю ты перестанешь замечать классы и начнешь видеть *структуру*.
Пока что мы используем SCSS (по старинке), но в новых компонентах пробуем внедрять Tailwind (по желанию).
