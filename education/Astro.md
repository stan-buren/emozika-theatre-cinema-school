# 🚀 Astro: Наш Двигатель

## Что это?
**Astro** — это фреймворк для создания веб-сайтов.
Если в CS50 ты использовал **Flask** (Python) для рендеринга HTML-шаблонов (`render_template("index.html")`), то Astro делает похожую работу, но на JavaScript.

## Главная фишка: "Islands" (Острова) 🏝
В современном вебе (React, Vue) часто грузится **огромный** кусок JavaScript, даже если он не нужен. Это тормозит сайт.

Astro работает иначе:
1.  Он берет твой код (`.astro` файлы).
2.  Во время сборки (Build) он выполняет весь JavaScript.
3.  На выходе отдает **чистый HTML** + CSS (как если бы ты написал его руками в Блокноте).
4.  JavaScript грузится *только* там, где есть интерактивность (например, наш фильтр в Студии или карта). Это и есть "Острова интерактивности" в море статического HTML.

## Аналогия с CS50
*   **C (clang):** Ты пишешь код, компилируешь его (`make`), получаешь бинарник. Бинарник работает быстро.
*   **Astro:** Ты пишешь компоненты, "компилируешь" их (`npm run build`), получаешь HTML. HTML грузится мгновенно.

## Структура файла `.astro`
Он выглядит как бутерброд:

```astro
---
// 1. Server Script (JavaScript)
// Этот код выполняется ТОЛЬКО во время сборки.
// Тут мы тянем данные, импортируем переменные.
// В браузере этого кода не будет!
const title = "Привет, CS50!";
---

<!-- 2. Template (HTML) -->
<!-- Тут мы используем переменные сверху -->
<h1>{title}</h1>

<style>
  /* 3. Styles (CSS) */
  /* Стили действуют ТОЛЬКО внутри этого файла (Scoped) */
  h1 { color: red; }
</style>
```

Это очень похоже на Jinja2 шаблоны во Flask (`{{ title }}`), только скобки одинарные `{ title }`.
